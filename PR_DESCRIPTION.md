## 작업 내용

- 정책 추천 온보딩 화면을 추가했습니다.
  - 회원가입 직후 `/policy-onboarding`으로 이동해 정책 추천 조건을 입력할 수 있습니다.
  - 나이, 영농 형태, 거주/농지 지역, 작물, 재배 면적, 신청자 유형, 관심 지원 분야, 기존 수혜 정책 등을 단계형 질문으로 수집합니다.
  - 입력 결과는 정책 온보딩 API 후보 경로에 `POST` 또는 `PUT`으로 저장합니다.

- 지원사업 페이지를 백엔드 정책 API 기반으로 전환했습니다.
  - 기존 더미 지원사업 목록 대신 정책 API 응답을 정규화해 표시합니다.
  - 검색, 카테고리/작물 필터, 마감일/지원금/출처 정렬을 지원합니다.
  - 정책 상세 모달에 출처, 신청 URL, 신청 자격, 지원 내용, 문의처를 표시합니다.
  - 즐겨찾기 상태는 기존 localStorage 흐름을 유지합니다.

- 대시보드에 날씨 기반 농장 브리핑 카드를 추가했습니다.
  - 기존 재배 작물 수 중심 카드 대신 `summary`, `actions`, `warnings` 중심 카드로 교체했습니다.
  - 메인 화면에서 농장 프로필, 재배 작물, 현재 날씨를 모아 브리핑 API로 전달합니다.
  - 기본 호출은 `forceRefresh=false`, 새로고침/지역 변경 직후에는 `forceRefresh=true`를 사용합니다.
  - 기본 브리핑 API 경로는 `/api/v1/weather-farm-briefing`이며 `VITE_FARM_BRIEFING_PATH`로 변경할 수 있습니다.

- 지역 선택 UI를 공용화했습니다.
  - 마이페이지의 직접 구현된 지역 선택 모달을 공용 `RegionPickerModal`로 교체했습니다.
  - 대시보드 브리핑 카드에서도 마이페이지와 같은 도/시군구 선택 방식을 사용합니다.
  - 지역 좌표/날씨 조회 로직은 공용 유틸로 분리했습니다.

## 테스트

- `npm run build` 성공
- 로컬 dev 서버 응답 확인

## 참고

- `src/components/AppLayout.vue`는 이번 작업에서 사용되지 않는 기존 미추적 파일이라 커밋에 포함하지 않았습니다.
